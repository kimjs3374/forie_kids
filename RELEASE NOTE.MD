## 1. 릴리즈 개요

v2.0은 기존 예약/입금확인 중심 운영 흐름을 유지하면서도, **문의사항 관리 체계 정비**, **월별 이용금액 운영 방식 개선**, **예약 완료 안내 강화**, **운영자용 관리 화면 개선**, **DB/운영 스크립트 보강**에 초점을 맞춘 업데이트입니다.

이번 버전에서는 특히 기존의 `payment_request` 개념을 사용자 친화적인 **문의사항(inquiry)** 중심으로 재정리했고, 예약 월마다 개별 금액을 관리할 수 있도록 구조를 변경했습니다. 또한 0원 예약 자동확정, 계좌번호 복사 기능, 관리자 답변/수정/삭제 기능 등 실제 운영 편의성이 크게 향상되었습니다.

## 2. 주요 변경사항 요약

### 2.1 사용자 기능 개선

- **문의사항 메뉴 신설/정비**
  - 기존 `payment_request` 흐름을 `inquiry` 기반으로 전환했습니다.
  - 사용자는 이름/연락처/동/호수와 함께 문의 내용을 등록할 수 있습니다.
  - 같은 화면에서 **답변조회**를 통해 내가 남긴 문의와 관리자 답변을 확인할 수 있습니다.

- **예약 완료 안내 강화**
  - 예약 성공 시 단순 성공 문구 대신, 아래 정보가 포함된 안내 모달을 제공합니다.
    - 신청 월
    - 동/호수
    - 신청자명
    - 아이 인원수
    - 입금 금액
    - 입금 계좌 정보
  - 계좌번호가 설정되어 있으면 **계좌번호 복사 버튼**으로 바로 복사할 수 있습니다.

- **0원 예약 자동 확정 지원**
  - 해당 월 이용금액이 `0원`이면 예약 접수 직후 자동으로 `PAYMENT_CONFIRMED` 처리됩니다.
  - 자동확정 시 텔레그램 알림도 함께 연동되도록 보완되었습니다.

- **입력 UX 개선**
  - 신청하기 / 신청조회 / 비밀번호 / 문의사항 화면의 입력 placeholder와 검증 메시지를 보다 직관적으로 정리했습니다.
  - 이름/동/호수 누락 시 사용자에게 더 명확한 오류 메시지가 노출됩니다.

### 2.2 관리자 기능 개선

- **문의사항 관리 화면 추가**
  - 관리자 메뉴에 `문의사항 관리` 기능이 추가되었습니다.
  - 사용자 문의 스레드를 최신순으로 확인할 수 있습니다.
  - 관리자 답변 등록 가능
  - 기존 문의/답변 **수정 가능**
  - 기존 문의/답변 **삭제 가능**

- **예약 월 관리 개선**
  - 예약 월 생성 시 **월별 이용금액**을 직접 입력할 수 있습니다.
  - 등록된 예약 월 목록 UI가 표 중심에서 **카드형 + 모달 수정 구조**로 개선되었습니다.
  - 월별로 다음 정보가 더 명확히 노출됩니다.
    - 이용금액
    - 정원
    - 입장 비밀번호
    - 오픈일 / 마감일

- **도움말/운영 가이드 업데이트**
  - 사용자 도움말과 관리자 도움말 문구가 v2.0 기능 기준으로 정비되었습니다.
  - 문의사항 흐름, 월별 금액, 0원 자동확정, 입금 처리 흐름 등의 설명이 반영되었습니다.

### 2.3 입금확인/은행연동 정책 개선

- **월별 이용금액 기준으로 운영 방식 변경**
  - 기존처럼 은행 설정에서 예약금액을 전역 관리하는 방식에서 벗어나,
    **예약 월마다 `payment_amount`를 관리**하도록 정리되었습니다.
  - 은행 설정은 계좌/연동 정보 중심으로 단순화되었습니다.

- **환경변수 기반 기본 계좌정보 fallback 추가**
  - 은행 설정 조회에 실패하거나 값이 없을 때 `.env` 기본값으로 계좌정보를 안내할 수 있습니다.
  - 추가 환경변수:
    - `BANK_DEFAULT_CODE`
    - `BANK_DEFAULT_ACCOUNT_HOLDER_NAME`
    - `BANK_DEFAULT_ACCOUNT_NUMBER`
    - `BANK_DEFAULT_PAYMENT_AMOUNT`

- **입금 수집 필터 강화**
  - 자동 수집 시 **입금(deposit) 거래만 저장**하도록 정리했습니다.
  - 또한 `30,000원 이하 입금`만 기록하도록 제한(`MAX_RECORDED_DEPOSIT_AMOUNT = 30000`)이 추가되었습니다.
  - 불필요한 거래 적재를 줄여 자동매칭 정확도와 운영 안정성을 높였습니다.

### 2.4 데이터 정비 및 백엔드 구조 개선

- **문의사항 데이터 모델로 명칭 통일**
  - `deposit_requests` → `inquiries`
  - `deposit_request_messages` → `inquiry_messages`
  - 코드, 템플릿, 정리 배치, 스키마 전반이 해당 명칭으로 정리되었습니다.

- **개인정보 정리 대상 변경 반영**
  - 개인정보 삭제 로직이 기존 `deposit_requests` 기준에서
    `inquiries / inquiry_messages` 기준으로 변경되었습니다.

- **예약 생성 RPC 보정**
  - `create_reservation_atomic` 함수가 alias 사용 및 반환 컬럼명 명시 등으로 정리되었습니다.
  - 운영 중 SQL 함수 정합성 문제를 줄이기 위한 보정 스크립트가 추가되었습니다.

- **앱 실행 구조 정리**
  - `run.py`가 `create_app()` 기반으로 정리되어 앱 팩토리 구조와의 일관성이 향상되었습니다.

## 3. 상세 변경 포인트

### 3.1 문의사항 기능 전환

이번 릴리즈에서 가장 큰 변화 중 하나는 기존의 입금확인 요청 성격의 화면을 **문의사항 게시판형 흐름**으로 정리한 점입니다.

- 사용자 라우트 변경
  - `payment.py` → `inquiry.py`
  - `payment_request.html` → `inquiry.html`
- 관리자 라우트 추가
  - `app/routes/admin_routes/inquiries.py`
- 관리자 템플릿 추가
  - `templates/admin/inquiries.html`
- 관리자 답변/수정/삭제 흐름 지원

이로 인해 사용자는 단순 요청 등록이 아니라, **문의와 답변 이력을 누적형으로 확인**할 수 있게 되었습니다.

### 3.2 월별 이용금액 운영

v2.0부터 이용금액은 은행설정의 전역값이 아니라 **예약 월 개별값**으로 관리됩니다.

- 새 예약 월 생성 시 금액 입력 가능
- 기존 월 수정 시 금액 수정 가능
- 월별 카드 UI에 금액 표시
- 메인 신청 화면 및 신청 완료 모달에 해당 월 금액 표시

이를 통해 프로모션 월, 무료 운영 월, 특정 월만 다른 요금 적용 같은 운영이 가능해졌습니다.

### 3.3 신청 완료 경험 개선

기존에는 예약 신청 완료 후 상태만 안내했다면, v2.0에서는 사용자가 실제로 바로 행동할 수 있도록 **입금 정보까지 포함한 완료 모달**을 제공합니다.

- 입금 금액 강조 표시
- 은행명/계좌번호/예금주 안내
- 계좌번호 복사 버튼 제공
- 복사가 어려운 환경을 위한 fallback 처리 포함

이 변경은 입금 지연, 계좌번호 오입력, 문의 증가를 줄이기 위한 목적을 가집니다.

## 4. 운영/배포 시 확인사항

### 4.1 DB 마이그레이션 관련

기존 v1.x 데이터를 유지한 채 업그레이드하는 경우, 아래 SQL 반영 여부를 반드시 확인해야 합니다.

- `supabase_schema.sql`
- `scripts/rename_inquiries.sql`
- `scripts/fix_create_reservation_atomic.sql`

필요 시 아래 신규 스크립트도 운영 목적에 따라 사용할 수 있습니다.

- `scripts/hard_reset_recreate_db.sql`
- `scripts/reset_all_data.sql`

### 4.2 환경변수 추가

`.env` 또는 운영 환경에 아래 값 추가를 권장합니다.

```env
BANK_DEFAULT_CODE=NH
BANK_DEFAULT_ACCOUNT_HOLDER_NAME=
BANK_DEFAULT_ACCOUNT_NUMBER=
BANK_DEFAULT_PAYMENT_AMOUNT=5000
```

### 4.3 운영 정책 변경점

- 문의 관련 테이블/용어가 `deposit_request`에서 `inquiry`로 바뀌었습니다.
- 월별 이용금액은 **은행연동 화면이 아니라 예약 월 관리 화면**에서 관리합니다.
- 0원 월은 신청 즉시 자동 완료 처리됩니다.
- 자동 입금 수집은 **입금(deposit) + 30,000원 이하** 조건으로만 적재됩니다.

## 8. 결론

v2.0은 단순 기능 추가를 넘어서, **운영자가 관리하기 쉬운 구조**와 **사용자가 이해하기 쉬운 흐름**으로 서비스를 재정비한 릴리즈입니다. 특히 문의사항 관리, 월별 이용금액, 0원 예약 처리, 신청 완료 안내 강화는 실제 운영 효율을 크게 높일 수 있는 핵심 개선사항입니다.

기존 v1.5에서 업그레이드하는 경우에는 **DB 스키마/스크립트 반영**과 **환경변수 추가**, 그리고 **운영자 가이드 재숙지**를 함께 진행하는 것을 권장합니다.
