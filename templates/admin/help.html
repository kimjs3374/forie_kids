관리자용 도움말

이 페이지는 관리자 로그인 상태에서만 볼 수 있으며, 상단 관리자 메뉴의 도움말 탭에서 다시 확인할 수 있습니다.

1. 대시보드

관리자 로그인 후 가장 먼저 보는 기본 운영 화면입니다.

대시보드 열기
  • 공지사항 관리 : 메인 화면 상단의 운영 공지 문구를 수정하고 저장합니다.
  • 자동 입금 확인 현황 : 은행연동 활성 상태, 자동매칭 수, 미매칭 수, 최근 오류를 빠르게 점검합니다.
  • 예약현황 : 월별 정원, 신청 수, 완료 수, 현재 상태를 한 번에 확인합니다.
  • 최근 예약 : 최신 접수 내역과 입금 확인 여부를 간단히 확인합니다.
  • 텔레그램 바로가기 : 상단 텔레그램 버튼을 누르면 새 창에서 운영용 텔레그램 링크로 바로 이동합니다. 버튼은 .env의 ADMIN_TELEGRAM_URL 값이 있을 때만 표시됩니다.

2. 공지 및 전광판 관리

메인 화면에 보이는 안내 문구를 관리하는 기능입니다.

메인 화면 열기
  • 운영 공지 : 대시보드에서 저장한 공지사항은 메인 화면 우측 안내 영역에 바로 노출됩니다.
  • 전광판 직접 관리 : 관리자 로그인 상태로 메인 화면에 접속하면 공지사항 & 입금자를 찾습니다 영역에 관리 버튼이 나타납니다.
  • 전광판 문구 추가/수정/삭제 : 문구 내용, 노출 시간, 정렬 순서를 조정할 수 있습니다.
  • BANK 전광판 : 은행 거래에서 전광판 승인된 건은 자동 생성 문구로 표시되며, 메인 화면 관리창 또는 은행연동 화면에서 승인 해제할 수 있습니다.

3. 예약생성

예약 월을 만들고 수정/삭제하는 메뉴입니다.

예약생성 열기
  • 예약 월 생성 : 연도, 월, 제목, 접수 시작일, 접수 마감일, 월별 이용금액, 정원을 입력해 새 예약 월을 만듭니다.
  • 자동 생성 기준 : 시스템은 매월 1일에 다음달 예약을 자동 생성하도록 설계되어 있습니다.
  • 이용금액 기본값 : 새 월 생성 시 이용금액 기본값은 5,000원이며, 월마다 다르게 저장할 수 있습니다. 0원도 허용됩니다.
  • 표시 방식 변경 : 등록된 예약 월은 표가 아니라 우측 리스트 카드 형태로 표시되며, 각 월의 이용금액/정원/비밀번호/오픈·마감일을 한눈에 볼 수 있습니다.
  • 수정/삭제 : 이미 등록된 월은 수정 버튼을 눌러 모달에서 변경할 수 있고, 필요하면 삭제도 가능합니다.

4. 예약관리

신청자 정보와 입금 상태를 확인하는 핵심 운영 메뉴입니다.

예약관리 열기
  • 필터 : 전체 / 입금완료 / 입금대기 기준으로 목록을 빠르게 좁힐 수 있습니다.
  • 입금 토글 : 각 예약의 입금확인 / 입금대기 상태를 버튼 하나로 전환할 수 있습니다.
  • 표시 정보 : 예약번호, 신청자, 동호수, 연락처, 금액, 상태, 입금확인일, 신청일시를 확인합니다.
  • 0원 예약 처리 : 이용금액이 0원인 월은 신청 즉시 자동으로 예약완료 처리됩니다.
  • 엑셀 다운로드 : 현재 예약 내역을 엑셀 파일로 내려받을 수 있습니다.

5. 은행연동

계좌 설정, 거래 동기화, 자동 매칭, 수동 매칭을 처리하는 메뉴입니다.

은행연동 열기
  • 계좌 연동 설정 : 은행 코드, 예금주, 계좌번호, 비밀번호, 생년월일/사업자번호를 저장합니다. 민감 정보는 암호화 저장됩니다.
  • 즉시 동기화 : 수동 실행 시 02:00~07:00 중지 시간과 관계없이 거래를 즉시 조회합니다.
  • 자동 매칭 기준 : 금액, 입금자명, 예약 이후 입금 여부를 보고 후보가 1건일 때 자동 연결합니다.
  • 수동 매칭 : 자동 매칭 실패 건은 후보 목록 또는 예약번호 직접 입력으로 연결할 수 있습니다.
  • 전광판 승인/무시 처리 : 거래를 전광판에 노출하거나, 운영상 제외가 필요한 거래는 무시 처리할 수 있습니다.

6. 문의사항 관리

사용자가 남긴 문의와 관리자 답변을 스레드 형태로 관리합니다.

문의사항 열기
  • 스레드 목록 : 사용자별 요청을 최근 업데이트 순으로 확인합니다.
  • 상세 열기 : 각 요청 박스를 펼치면 사용자 요청 내용과 기존 답변을 모두 확인할 수 있습니다.
  • 답변 등록 : 답변 내용을 입력하고 등록하면 해당 스레드에 누적 저장됩니다.
  • 문의/답변 수정 : 각 사용자 요청/관리자 답변마다 수정 버튼이 있어 모달에서 내용을 바로 고칠 수 있습니다.
  • 문의/답변 삭제 : 잘못 등록된 내용은 삭제 버튼으로 제거할 수 있습니다. 마지막 메시지까지 삭제하면 해당 문의 스레드 자체도 함께 정리됩니다.
  • 사용자 확인 : 사용자는 같은 화면의 답변조회 기능으로 관리자 답변을 확인합니다.

7. 비번관리

월별 입장 비밀번호를 생성하고 저장하는 메뉴입니다.

비번관리 열기
  • 랜덤생성 : 기존과 겹치지 않는 4자리 숫자를 자동 생성해 입력칸에 넣을 수 있습니다.
  • 저장 반영 : 랜덤생성만으로는 실제 반영되지 않으며, 반드시 저장 버튼을 눌러야 적용됩니다.
  • 사용자 노출 조건 : 이번 달 예약완료 이용자만 비밀번호 조회 메뉴에서 확인할 수 있습니다.

8. 운영 팁 및 참고사항

실제 운영 시 자주 확인하는 포인트입니다.

  • 도움말 노출 방식 : 사용자용 도움말은 모든 페이지 하단 도움말 링크에서 볼 수 있고, 관리자용 도움말은 로그인 후 상단 관리자 메뉴에서만 보입니다.
  • 입금 처리 순서 : 자동 매칭이 안 된 건은 먼저 은행연동에서 확인한 뒤, 필요하면 예약관리/문의사항과 함께 대조하는 것이 좋습니다.
  • 전광판 운영 : 실제 공지성 문구와 입금자 찾기 문구가 함께 노출될 수 있으므로, 오래된 내용은 정리해두는 것이 좋습니다.
  • 텔레그램 버튼 설정 : 관리자 상단 텔레그램 버튼은 .env에 ADMIN_TELEGRAM_URL=https://t.me/... 형식으로 추가하면 활성화됩니다. 변경 후에는 앱을 재시작해 반영하세요.
  • 로그아웃 : 상단 로그아웃 버튼으로 세션을 종료하면 관리자 전용 메뉴와 도움말은 더 이상 보이지 않습니다.
